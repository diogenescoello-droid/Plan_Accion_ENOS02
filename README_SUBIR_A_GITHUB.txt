ENOS Guayas 2.1 - Paquete final

Contenido principal:
- HTML modular actualizado por instituciones, ejes del plan, cartografía, dashboard, automatización, GPT, biblioteca, accesos y validación.
- Carpeta docs con documentos base, insumos SPREA, formularios XLSForm y matrices.
- Carpeta assets/img con gráficos.
- Carpeta automatizacion con plantilla conceptual de Apps Script.

Cómo publicar en GitHub Pages:
1. Descomprimir este ZIP.
2. Subir todo el contenido interno al repositorio, al mismo nivel de index.html.
3. No cambiar nombres de carpetas: assets, docs, data, automatizacion.
4. Verificar que index.html cargue.
5. Probar biblioteca, formularios Kobo y GPT.
6. Publicar F10 y F11 en Kobo cuando se vaya a activar solicitud de acceso y propuestas de escenarios.
7. Conectar dashboard real cuando exista base consolidada automática.

Advertencia:
La plataforma pública no debe exponer enlaces administrativos de Kobo ni carpetas con datos sensibles.
